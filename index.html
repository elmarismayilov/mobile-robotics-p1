DEPLOYMENT PROTOCOL

How to Run the AMNR System

A streamlined operational flow to transition from cold hardware to an interactive, mapping autonomous agent.

âš¡ Core Requirements

  • Hardware: TurtleBot3 Burger + Host Laptop
  • Network: Shared local network via Ethernet Cable
  • OS: Ubuntu 22.04 + ROS2 Humble
  • State: Fully charged battery (above 11.1V)
STEP 01

Connectivity & Sync

The foundation of the distributed system is a stable, high-bandwidth Ethernet link between the Raspberry Pi and the Host PC.

  • 1

    Physical Link

    Connect the Ethernet cable from the Laptop to the Burger's Pi 4 board.

  • 2

    IP Verification

    Ensure both devices are on the same subnet (e.g., 192.168.1.xxx).

  • 3

    ROS Domain ID

    Export matching ROS_DOMAIN_ID on both terminals to enable DDS discovery.

$export ROS_DOMAIN_ID=30
$ping 192.168.1.100
# Success: 0% packet loss
STEP 02

The Edge Bringup

Initiate the robot's primary sensor drivers and the OpenCR communication node from the Raspberry Pi ssh terminal.

ðŸ“¡ Distributed Nodes

The Pi starts the LIDAR scan and IMU publisher nodes. These high-frequency topics are broadcast over Ethernet to the PC.

# From Laptop terminal (SSH)
$ssh pi@192.168.1.100
$ros2 launch turtlebot3_bringup robot.launch.py

[INFO] [LDS]: LDS-02 is rotating...
[INFO] [OpenCR]: Micro-controller connected.

STEP 03

Host Orchestration

Run the heavy computational layers on the Host PC. This starts the SLAM engine and the Navigation stack.

  • 1

    SLAM Toolbox

    Loads the asynchronous mapping node to build the occupancy grid.

  • 2

    Nav2 Stack

    Initializes global/local planners and recovery behaviors.

# Open two terminals on Host PC
$ros2 launch slam_toolbox online_async_launch.py
$ros2 launch turtlebot3_navigation2 navigation2.launch.py
STEP 04

Vision & Interaction

Use RViz2 on your Host PC to visualize the livedataset and send coordinate goals to the robot.

ðŸŽ¯ Goal Setting

1. Open RViz2
2. Select the "2D Nav Goal" tool (keyboard shortcut: G)
3. Click and drag a direction on the live map.
4. TurtleBot3 will plan a path and execute.

$rviz2 -d $(ros2 pkg prefix turtlebot3_navigation2)/share/turtlebot3_navigation2/rviz/turtlebot3_navigation2.rviz

PRO TIP

The map updates in real-time as the robot moves. If the path turns red, it means a static obstacle is in the way.
